PHASE 0 ── 事前準備（セッション開始前）
STEP 0 ca.crt を受け取り・保存する 💻 あなた
サーバーから事前に送られてくる ca.crt を保存する。 これはブラウザやOSに組み込まれたルートCA証明書ストアに相当する。 サーバーが当日送ってくる証明書はこのCAで署名されているので、これを信頼の起点にする。
bash — 作業ディレクトリを作って保存
mkdir tls-demo && cd tls-demo

# サーバーから受け取った ca.crt を書き出す（BEGIN〜END ごと）
cat > ca.crt  # 貼り付けて Ctrl+D

# 確認（Subject が TLS-Theater-CA になっていれば OK）
openssl x509 -in ca.crt -noout -subject -dates
PHASE 1 ── ClientHello
STEP 1 ClientRandom 生成 → 投稿 💻 あなた
32バイトの乱数を生成する。これが ClientHello の核心。
bash
openssl rand -hex 32

Discordに投稿する
ClientHello です。 対応暗号スイート: TLS_RSA_WITH_AES_128_CBC_SHA256 圧縮: null ClientRandom: <出力した 64文字の hex>
PHASE 2 ── ServerHello / Certificate 受信
⏳
サーバー待ち — ServerHello（ServerRandom）、Certificate（server.crt）、ServerHelloDone の3つが投稿されるまで待つ
STEP 2 ServerRandom を変数に保存 💻 あなた
Discord に投稿された ServerRandom（hex 64文字）をコピーして変数に入れておく。
bash
SERVER_RANDOM="（Discord からコピーした 64文字）"
STEP 3 証明書を検証 → 公開鍵を取り出す 💻 あなた
Discord に貼られた -----BEGIN CERTIFICATE----- 〜 -----END CERTIFICATE----- を保存し、事前に受け取った ca.crt で署名を検証する。これがブラウザの「錠前マーク」に相当する。
bash
# Discord の証明書を書き出す（BEGIN〜END ごと）
cat > server.crt  # 貼り付けて Ctrl+D

# CA で署名を検証する（← ここがブラウザの証明書検証）
openssl verify -CAfile ca.crt server.crt
# → server.crt: OK  と出れば本物のサーバー

# 公開鍵を取り出す
openssl x509 -in server.crt -pubkey -noout > server_pubkey.pem
PHASE 3 ── ClientKeyExchange（Pre-Master Secret）
STEP 4 PMS 生成 → 暗号化 → 投稿 💻 あなた
ここが TLS の核心。先頭 2バイト 0303（TLS 1.2 バージョン）+ 46バイト乱数 = 48バイトの Pre-Master Secret を、サーバーの公開鍵で暗号化する。
bash
# PMS 生成（先頭 2バイト = 0303 ＋ 乱数 46バイト）
printf '\x03\x03' > pms.bin
openssl rand 46 >> pms.bin

# 先頭が 03 03 か確認
od -An -tx1 -N2 pms.bin
# →  03 03

# サーバー公開鍵で RSA 暗号化
openssl pkeyutl -encrypt \
  -inkey server_pubkey.pem -pubin \
  -in    pms.bin \
  -out   enc_pms.bin

# base64 に変換して Discord に貼る
base64 -b 0 enc_pms.bin
💬
Discordに投稿する
ClientKeyExchange です。 PMS をサーバーの公開鍵で暗号化しました。 server.key なしには復号不可能です 🔐 <base64 の出力をそのまま貼る>

使い方

python3 prf.py <pms_hex> <client_random> <server_random>

# Finished verify_data 生成（STEP 7 で使う）

STEP 6 セッション鍵を導出する 💻 あなた
PMS・両乱数を使って鍵素材を展開する。出力は絶対に Discord に貼らない。
bash
# PMS を hex 文字列に変換
PMS_HEX=$(od -An -tx1 pms.bin | tr -d ' \n')

CLIENT_RANDOM="（STEP 1 で生成した 64文字）"
SERVER_RANDOM="（STEP 2 で保存した 64文字）"

python3 prf.py $PMS_HEX $CLIENT_RANDOM $SERVER_RANDOM
🔑 出力の client_write_key / client_write_iv / client_write_MAC / master_secret を手元に控えておく。Discordには投稿しない。
PHASE 5 ── ChangeCipherSpec & Finished
STEP 7 Client Finished を暗号化 → 投稿 💻 あなた
ここまでの Discord ログをそのままハッシュ化して client_write_key で暗号化する。サーバーが復号できれば「同じ鍵 ＆ 同じログを見ていた（改ざんなし）」の証明になる。
bash
# Discord のログを貼り付ける（ClientHello から ServerHelloDone まで）
# ※サーバー役と全く同じ内容をコピペすること
cat > hs_log.txt  # 貼り付けて Ctrl+D

# ハッシュ化
HS_HASH=$(openssl dgst -sha256 -hex hs_log.txt | cut -d' ' -f2)

MS_HEX="（master_secret の値）"
CW_KEY="（client_write_key の値）"
CW_IV="（client_write_iv の値）"

# verify_data = PRF(MS, "client finished", HS_HASH)[12B] → 暗号化
python3 prf.py finished $MS_HEX $HS_HASH client | \
  python3 -c "import sys; sys.stdout.buffer.write(bytes.fromhex(input()))" | \
  openssl enc -aes-128-cbc \
  -K $CW_KEY -iv $CW_IV -nosalt | \
  base64 -b 0
⚠️ サーバー役と全く同じ内容をコピペしないとハッシュが一致しない。空白・改行の差も影響するので、貼る範囲を事前に打ち合わせておく。
💬
Discordに投稿する
ChangeCipherSpec です。以降は暗号化します。 Finished（暗号文）: <base64 の出力をそのまま貼る>
⏳
サーバー待ち — サーバーの ChangeCipherSpec + Finished が投稿されるのを待つ
STEP 8 Server Finished を検証する 💻 あなた
server_write_key で復号し、12バイトの verify_data が出ればハンドシェイク完了。
bash
SW_KEY="（server_write_key の値）"
SW_IV="（server_write_iv の値）"

echo "（サーバーの base64）" | base64 -d | \
  openssl enc -d -aes-128-cbc \
  -K $SW_KEY -iv $SW_IV -nosalt | \
  od -An -tx1
# → 12バイト（24文字）表示されれば OK
PHASE 6 ── Application Data（秘密の通信）
STEP 9 メッセージを暗号化 → 投稿 💻 あなた
平文に HMAC-SHA256 を付けてから AES-128-CBC で暗号化する。
bash
MSG="ランチは12時に渋谷で"
echo -n "$MSG" > msg.txt
CW_MAC="（client_write_MAC の値）"

# MAC = HMAC-SHA256(CW_MAC, SeqNum(8B) || 平文)
printf '\x00\x00\x00\x00\x00\x00\x00\x00' > seq.bin
cat seq.bin msg.txt | \
  openssl dgst -sha256 -mac HMAC \
  -macopt "hexkey:$CW_MAC" -binary > mac.bin

# 平文 + MAC を連結して暗号化
cat msg.txt mac.bin | \
  openssl enc -aes-128-cbc \
  -K $CW_KEY -iv $CW_IV -nosalt | \
  base64 -b 0
