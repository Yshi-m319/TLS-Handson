TLS Discord Theater — Command Reference

🖥️ サーバー役マニュアル

openssl / python3 / macOS
あなたの操作 両者（各自実行） 待機 Discordに投稿
PHASE 0 ── 事前準備（セッション前）
STEP 0-a サーバー秘密鍵と CSR を生成する 🖥️ あなた
🔒 server.key は絶対に公開しない — これが唯一の秘密
bash
mkdir tls-demo && cd tls-demo

# サーバー秘密鍵
openssl genrsa -out server.key 2048

# CSR（証明書署名要求）を作成
openssl req -new \
  -key server.key \
  -out server.csr \
  -subj "/CN=TLS-Theater-Server"

# CSR の中身を確認
openssl req -in server.csr -noout -subject
STEP 0-b CSR を CA役（主催者）に送る 🖥️ あなた
CSR は秘密情報ではない（秘密鍵は含まれない）。DM やチャンネルで CA役に送って署名してもらう。
bash
cat server.csr
💬
CA役（主催者）に送る
CSR です。署名をお願いします。 -----BEGIN CERTIFICATE REQUEST----- <cat の出力をそのまま貼る> -----END CERTIFICATE REQUEST-----
⏳
CA役待ち — 署名済み server.crt が返ってくるまで待つ
STEP 0-c server.crt を受け取り・保存する 🖥️ あなた
CA役から返ってきた -----BEGIN CERTIFICATE----- 〜 -----END CERTIFICATE----- を保存する。
bash
# 署名済み証明書を書き出す（BEGIN〜END ごと）
cat > server.crt  # 貼り付けて Ctrl+D

# 内容を確認
openssl x509 -in server.crt -noout -subject -issuer -dates
PHASE 1 ── ClientHello 受信
⏳
クライアント待ち — ClientHello（ClientRandom）が投稿されるのを待つ
STEP 1 ClientRandom を変数に保存 🖥️ あなた
Discord に投稿された ClientRandom（hex 64文字）を変数に入れておく。
bash
CLIENT_RANDOM="（Discord からコピーした 64文字）"
PHASE 2 ── ServerHello / Certificate / ServerHelloDone 送信
STEP 2 ServerRandom 生成 → ServerHello 投稿 🖥️ あなた
乱数を生成し、使う暗号スイートを決めて投稿する。
bash
openssl rand -hex 32
💬
Discordに投稿する
ServerHello です。 選択した暗号スイート: TLS_RSA_WITH_AES_128_CBC_SHA256 圧縮: null ServerRandom: <出力した 64文字の hex>
STEP 3 server.crt を Certificate として投稿 🖥️ あなた
CA に署名された server.crt をそのまま貼る。クライアントはこれを ca.crt で検証してから公開鍵を取り出す。
bash
# フィンガープリント（改ざん確認用）
openssl x509 -in server.crt -fingerprint -sha256 -noout

# 証明書本体
cat server.crt
💬
Discordに投稿する
Certificate です。 Fingerprint: <SHA256:xx:xx:...> -----BEGIN CERTIFICATE----- <cat の出力をそのまま貼る> -----END CERTIFICATE-----
STEP 4 ServerHelloDone 投稿 🖥️ あなた
コマンドなし。テキストだけ投稿する。
💬
Discordに投稿する
ServerHelloDone です。あとはお任せします。
PHASE 3 ── ClientKeyExchange 受信・PMS 復号
⏳
クライアント待ち — ClientKeyExchange（暗号化PMS）が投稿されるのを待つ
STEP 5 PMS を秘密鍵で復号する 🖥️ あなた
🔒 この操作は Discord に投稿しない
先頭 2バイトが 03 03 なら正しい PMS。
bash
# Discord の base64 をデコードして保存
echo "（base64 をここに貼る）" | base64 -d > enc_pms.bin

# server.key で RSA 復号
openssl pkeyutl -decrypt \
  -inkey server.key \
  -in    enc_pms.bin \
  -out   pms.bin

# 先頭 2バイトが 03 03（TLS 1.2）か確認
od -An -tx1 -N2 pms.bin
# →  03 03
PHASE 4 ── 鍵導出（手元のみ・Discord には流さない）
STEP 6 prf.py を作業ディレクトリに置く 🤝 両者共通
配布された prf.py を tls-demo/ ディレクトリに置く。
bash
cp ~/Downloads/prf.py .
ls prf.py  # ← 表示されれば OK
使い方
# 鍵導出（STEP 7 で使う）
python3 prf.py <pms_hex> <client_random> <server_random>

# Finished verify_data 生成（STEP 9 で使う）
python3 prf.py finished <master_secret> <hs_hash> server
STEP 7 セッション鍵を導出する 🖥️ あなた
🔒 出力は絶対に Discord に貼らない
クライアントと同じ計算式で、独立して同じ鍵が出る。
bash
# PMS を hex 文字列に変換
PMS_HEX=$(od -An -tx1 pms.bin | tr -d ' \n')

SERVER_RANDOM="（STEP 2 で生成した 64文字）"
CLIENT_RANDOM="（STEP 1 で保存した 64文字）"

python3 prf.py $PMS_HEX $CLIENT_RANDOM $SERVER_RANDOM
🔑 出力の server_write_key / server_write_iv / client_write_key / client_write_iv / client_write_MAC / master_secret を手元に控えておく。
PHASE 5 ── ChangeCipherSpec & Finished
⏳
クライアント待ち — Client Finished（暗号文）が投稿されるのを待つ
STEP 8 Client Finished を検証する 🖥️ あなた
クライアントと同じ Discord ログをハッシュ化し、期待値を計算してから復号・照合する。
bash
# クライアントと同じ範囲の Discord ログを貼り付ける
cat > hs_log.txt  # 貼り付けて Ctrl+D

HS_HASH=$(openssl dgst -sha256 -hex hs_log.txt | cut -d' ' -f2)
MS_HEX="（master_secret の値）"
CW_KEY="（client_write_key の値）"
CW_IV="（client_write_iv の値）"

# 期待値を計算
EXPECTED=$(python3 prf.py finished $MS_HEX $HS_HASH client)

# クライアントの Finished を復号して照合
RECEIVED=$(echo "（クライアントの base64）" | base64 -d | \
  openssl enc -d -aes-128-cbc \
  -K $CW_KEY -iv $CW_IV -nosalt | \
  python3 -c "import sys; print(sys.stdin.buffer.read().hex())")

[ "$RECEIVED" = "$EXPECTED" ] \
  && echo "✓ Finished 検証 OK" \
  || echo "✗ 不一致 — ログの貼り付け範囲を確認"
STEP 9 Server Finished を送る 🖥️ あなた
同じ hs_log.txt を使って server 側の verify_data を作り、server_write_key で暗号化して投稿する。
bash
SW_KEY="（server_write_key の値）"
SW_IV="（server_write_iv の値）"

# verify_data = PRF(MS, "server finished", HS_HASH)[12B] → 暗号化
python3 prf.py finished $MS_HEX $HS_HASH server | \
  python3 -c "import sys; sys.stdout.buffer.write(bytes.fromhex(input()))" | \
  openssl enc -aes-128-cbc \
  -K $SW_KEY -iv $SW_IV -nosalt | \
  base64 -b 0
⚠️ STEP 8 で作った hs_log.txt と HS_HASH をそのまま使う。cat し直す必要はない。
💬
Discordに投稿する
ChangeCipherSpec です。 Finished（暗号文）: <base64 の出力をそのまま貼る> ハンドシェイク完了です 🎉
PHASE 6 ── Application Data 受信・復号
⏳
クライアント待ち — 暗号化メッセージが投稿されるのを待つ
STEP 10 復号 → MAC 検証 → 結果投稿 🖥️ あなた
AES 復号後、末尾 32バイトが HMAC なので切り離して再計算し、一致を確認する。
bash
CW_MAC="（client_write_MAC の値）"

# AES 復号
echo "（base64）" | base64 -d | \
  openssl enc -d -aes-128-cbc \
  -K $CW_KEY -iv $CW_IV -nosalt > dec.bin

# 末尾 32バイトが MAC、残りが平文
TOTAL=$(wc -c < dec.bin)
MLEN=$(( TOTAL - 32 ))
dd if=dec.bin bs=1 count=$MLEN  of=msg.bin  2>/dev/null
dd if=dec.bin bs=1 skip=$MLEN   of=rmac.bin 2>/dev/null

# 平文を表示
cat msg.bin

# MAC を再計算して検証
printf '\x00\x00\x00\x00\x00\x00\x00\x00' > seq.bin
RECV=$(od -An -tx1 rmac.bin | tr -d ' \n')
CALC=$(cat seq.bin msg.bin | \
  openssl dgst -sha256 -mac HMAC \
  -macopt "hexkey:$CW_MAC" -hex | cut -d' ' -f2)
[ "$RECV" = "$CALC" ] && echo "✓ MAC OK" || echo "✗ 改ざん検知"
💬
Discordに投稿する
復号できました 🎉 MAC検証: ✓（改ざんなし） メッセージ:「<cat の出力>」 全パケット傍受されてましたよね？ でも server.key なしには無理でしたよね 😏
