PHASE 0 ── CA鍵・ルート証明書の生成（セッション前）
STEP 1 CA 秘密鍵と自己署名証明書を生成する 🏛️ あなた
🔒 ca.key は絶対に公開しない — これが信頼の根拠
ca.key が漏洩すると攻撃者が偽のサーバー証明書を発行できる。 実世界の DigiCert や Let's Encrypt が厳重に保管しているのと同じ秘密。
bash
mkdir tls-ca && cd tls-ca

# CA 秘密鍵
openssl genrsa -out ca.key 2048

# CA 自己署名証明書（有効期限 1日）
openssl req -new -x509 -days 1 \
  -key ca.key -out ca.crt \
  -subj "/CN=TLS-Theater-CA"

# 内容を確認
openssl x509 -in ca.crt -noout -subject -dates
PHASE 1 ── サーバーの CSR に署名して server.crt を発行する
⏳
サーバー役待ち — サーバー役が server.csr を作成して送ってくるまで待つ
STEP 2 CSR を受け取り・保存する 🏛️ あなた
サーバー役から送られてきた -----BEGIN CERTIFICATE REQUEST----- 〜 -----END CERTIFICATE REQUEST----- を保存する。
bash
# CSR を書き出す（BEGIN〜END ごと）
cat > server.csr  # 貼り付けて Ctrl+D

# CSR の内容を確認（CN が TLS-Theater-Server になっていれば OK）
openssl req -in server.csr -noout -subject
STEP 3 CSR に署名して server.crt を発行する 🏛️ あなた
ca.key で署名することで「このサーバーはCAが認めた正規のサーバーである」という証明書が完成する。 これが実世界の証明書発行プロセスそのもの。
bash
openssl x509 -req -days 1 \
  -in     server.csr \
  -CA     ca.crt \
  -CAkey  ca.key \
  -CAcreateserial \
  -out    server.crt

# 署名が通っているか確認
openssl verify -CAfile ca.crt server.crt
# → server.crt: OK

# フィンガープリント（参加者への確認用）
openssl x509 -in server.crt -fingerprint -sha256 -noout
STEP 4 server.crt をサーバー役に返す 🏛️ あなた
署名済み証明書をサーバー役に渡す。セッション当日に Discord に貼るのはサーバー役の仕事。
bash
cat server.crt
💬
DM などでサーバー役に送る
署名しました。これがあなたの証明書です。 当日の Certificate ステップでこれを Discord に貼ってください。 -----BEGIN CERTIFICATE----- <cat の出力をそのまま貼る> -----END CERTIFICATE-----
PHASE 2 ── ca.crt を全参加者に配布する
STEP 5 ca.crt を全員に配布する 🏛️ あなた
ca.crt は秘密情報ではなく、ブラウザやOSに組み込まれたルートCA証明書ストアに相当する。 セッション前に Discord の全員が見えるチャンネルに貼ってよい。 クライアント役はこれを使ってサーバー証明書を検証する。
bash
cat ca.crt
💬
全員が見えるチャンネルにセッション前に投稿する
【事前配布】今日のセッションで使うルートCA証明書です。 クライアント役の方はこれを ca.crt として保存してください。 ブラウザの組み込みルートCAに相当します（ca.key は私だけが持っています）。 -----BEGIN CERTIFICATE----- <cat の出力をそのまま貼る> -----END CERTIFICATE-----
💡 観客への解説ポイント：実際のブラウザには数百のルートCA証明書が最初から組み込まれている。 今日の ca.crt はそのうちの1つに相当する。 これを信頼するということは「このCAが署名した証明書は全部信頼する」ということ。
PHASE 3 ── セッション当日（MC・進行役として）
STEP 6 当日は進行・解説に徹する 🏛️ あなた
当日の CA の仕事は以上で完了している。あとはサーバー役・クライアント役がハンドシェイクを進める。 主催者は MC として各ステップの意味を観客に解説する役割に徹する。
🎤 Certificate ステップでの解説例：
「サーバーが証明書を送りました。クライアントは今、この証明書が本当に私（CA）の署名を持っているかを openssl verify で確認しています。 ブラウザが錠前マークを表示するのはまさにこの検証が通ったときです」
